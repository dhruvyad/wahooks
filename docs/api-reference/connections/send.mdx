---
title: "Send Message"
api: "POST /api/connections/{id}/send"
description: "Send a text message through a WhatsApp connection."
---

Sends a text message to a specified chat through the WhatsApp connection. The connection must be in `online` status. The endpoint simulates natural typing behavior (marking the chat as seen, showing typing indicator) before sending the message to reduce spam detection risk.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token from Supabase Auth. Format: `Bearer <token>`
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the connection.
</ParamField>

## Body Parameters

<ParamField body="chatId" type="string" required>
  The WhatsApp chat ID to send the message to. Individual chats use the format `1234567890@c.us`, group chats use `1234567890-group@g.us`.
</ParamField>

<ParamField body="text" type="string" required>
  The text content of the message to send.
</ParamField>

<ResponseExample>
```json 200
{
  "id": "true_1234567890@c.us_3EB0A0B1C2D3E4F5",
  "timestamp": 1710410700
}
```

```json 400
{
  "statusCode": 400,
  "message": "chatId and text are required"
}
```

```json 401
{
  "statusCode": 401,
  "message": "Unauthorized"
}
```

```json 404
{
  "statusCode": 404,
  "message": "Connection not found"
}
```
</ResponseExample>
