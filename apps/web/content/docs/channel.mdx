The WAHooks channel lets you interact with [Claude Code](https://code.claude.com) directly from WhatsApp. Send a message on WhatsApp, and it appears in your Claude Code session. Claude's replies are sent back to you on WhatsApp.

## How it works

```
You (WhatsApp) → WAHA → WAHooks API → WebSocket → Channel → Claude Code
Claude Code    → wahooks_reply tool  → WAHooks API → WAHA  → You (WhatsApp)
```

The channel runs as a local process alongside Claude Code. It connects to the WAHooks API via WebSocket for real-time message delivery — no webhooks, tunnels, or port forwarding needed.

## Prerequisites

- A [WAHooks account](https://wahooks.com/signup) with a connected WhatsApp number
- [Claude Code](https://code.claude.com) v2.1.80+
- Node.js 18+
- [WAHooks CLI](/docs/cli/installation)

## Quick setup (recommended)

Two commands to get started:

```bash
wahooks claude setup    # login, install channel, create token, register MCP server
wahooks claude          # launch Claude Code with WhatsApp active
```

That's it. Run `wahooks claude setup` once, then `wahooks claude` anytime you want to chat via WhatsApp.

`wahooks claude setup` handles everything automatically:
1. Opens browser for authentication (skipped if already logged in)
2. Creates a dedicated API token for the channel
3. Installs `@wahooks/channel` via npm (skipped if already installed)
4. Saves channel config to `~/.claude/channels/wahooks/.env`
5. Registers the MCP server in `~/.claude.json`
6. Detects your active WhatsApp connection

After setup, `wahooks claude` launches Claude Code with the channel active and permissions bypassed so WhatsApp users get immediate responses.

## Manual setup

If you prefer to configure manually:

### 1. Install and configure

```bash
npm install -g @wahooks/channel
wahooks-channel --configure wh_your_api_key
```

### 2. Register with Claude Code

```bash
claude mcp add -s user wahooks-channel wahooks-channel
```

Or add manually to `~/.claude.json`:

```json
{
  "mcpServers": {
    "wahooks-channel": {
      "type": "stdio",
      "command": "wahooks-channel",
      "args": [],
      "env": {}
    }
  }
}
```

### 3. Start Claude Code

```bash
wahooks claude
```

Or manually:
```bash
claude --dangerously-load-development-channels server:wahooks-channel
```

You should see:
```
[wahooks-channel] Using connection: abc123...
[wahooks-channel] Webhook server listening on http://127.0.0.1:8790
[wahooks-channel] Ready — WhatsApp messages will appear in Claude Code
```

### 4. Send a WhatsApp message

Message your connected WhatsApp number. It will appear in Claude Code as:

```xml
<channel source="wahooks-channel" from="1234567890" message_id="msg_1">
list the files in the current directory
</channel>
```

Claude will process your request and reply via WhatsApp.

## Configuration

All configuration is stored in `~/.claude/channels/wahooks/.env` or can be set via environment variables:

| Variable | Description | Default |
|----------|-------------|---------|
| `WAHOOKS_API_KEY` | Your WAHooks API token (`wh_...`) | Required |
| `WAHOOKS_API_URL` | API base URL | `https://api.wahooks.com` |
| `WAHOOKS_CONNECTION` | Connection ID to use | Auto-detected if only one |
| `WAHOOKS_ALLOW` | Comma-separated allowed phone numbers | All (no filtering) |

## Sender gating

For security, restrict which phone numbers can send messages to Claude:

```bash
export WAHOOKS_ALLOW=1234567890,9876543210
```

Or add to your config file:
```
WAHOOKS_ALLOW=1234567890,9876543210
```

Messages from unlisted numbers are silently dropped.

## Permission relay

The channel supports [permission relay](https://code.claude.com/docs/en/channels-reference) — when Claude wants to run a tool (Bash, Write, etc.), it forwards the approval prompt to you on WhatsApp:

```
🔐 Claude wants to run: Bash
Execute: git status

Reply "yes abcde" to allow or "no abcde" to deny
```

Reply with the verdict and Claude continues.

## Available tools

When the channel is active, Claude has these tools:

| Tool | Description |
|------|-------------|
| `wahooks_reply` | Reply to the sender. Supports `reply_to` to quote a specific message. |
| `wahooks_react` | React to a message with an emoji (e.g. 👍, ❤️, 😂) |
| `wahooks_send` | Send a text message to any phone number |
| `wahooks_send_image` | Send an image (by URL or local `file_path`) with optional caption |
| `wahooks_send_document` | Send a document/file (by URL or local `file_path`) with optional filename |
| `wahooks_send_video` | Send a video (by URL or local `file_path`) with optional caption |
| `wahooks_send_audio` | Send an audio/voice message (by URL or local `file_path`) |
| `wahooks_send_location` | Send a location pin (latitude, longitude, name, address) |
| `wahooks_send_contact` | Send a contact card (name, phone number) |

## Multimodal conversations

The channel supports sending and receiving media (images, documents, videos, audio).

### Receiving media

When someone sends you a photo, video, or file on WhatsApp, the channel message includes the media URL. Claude can see the media type and download the file if needed:

```xml
<channel source="wahooks-channel" from="1234567890" message_id="msg_2">
Check this photo
[media: image/jpeg https://api.wahooks.com/api/connections/.../media/photo.jpeg]
</channel>
```

The media URL points to the WAHooks media proxy, which streams the file from the internal WAHA worker. The channel authenticates automatically.

### Sending media

Claude can send media back using the send tools. Each media tool accepts either a public URL or a local `file_path`:

```
# By URL
wahooks_send_image(chat_id, url="https://example.com/chart.png", caption="Here's the chart")

# By local file path
wahooks_send_image(chat_id, file_path="/tmp/screenshot.png", caption="Screenshot attached")
wahooks_send_document(chat_id, file_path="./report.pdf", filename="report.pdf")
```

This means Claude can generate a file locally (e.g. a chart, a code snippet, a PDF) and send it directly to you on WhatsApp.

## Troubleshooting

**No messages appearing in Claude Code**
- Check the channel connected: look for `[wahooks-channel] Connected to event stream` in output
- Make sure your WhatsApp connection is in `connected` status
- Check sender gating: if `WAHOOKS_ALLOW` is set, verify the sender's number is listed

**"No connected WAHooks connections found"**
- Create a connection and scan the QR code first
- Or set `WAHOOKS_CONNECTION` explicitly

**"No API key found"**
- Run `wahooks-channel --configure wh_your_key`
- Or set `WAHOOKS_API_KEY` environment variable
