The WAHooks MCP server lets AI assistants manage WhatsApp connections, send messages, and configure webhooks. It uses OAuth 2.1 with PKCE ā€” your browser opens once to authorize, and the token is saved automatically.

## Claude Code

Add to `.mcp.json` in your project root:

```json
{
  "mcpServers": {
    "wahooks": {
      "type": "http",
      "url": "https://api.wahooks.com/mcp"
    }
  }
}
```

Or use the CLI:

```bash
claude mcp add --transport http wahooks https://api.wahooks.com/mcp
```

<Callout type="info">
  Project-scoped config (`.mcp.json`) is recommended so your team can share the same setup via version control.
</Callout>

On first use, Claude Code will open your browser to authorize with your WAHooks account.

## Cursor

Add to `.cursor/mcp.json` in your project root (or globally at `~/.cursor/mcp.json`):

```json
{
  "mcpServers": {
    "wahooks": {
      "url": "https://api.wahooks.com/mcp"
    }
  }
}
```

Restart Cursor after saving. On first tool use, a browser window will open for OAuth authentication.

## Windsurf

Add to your Windsurf MCP config:

- **macOS/Linux**: `~/.codeium/windsurf/mcp_config.json`
- **Windows**: `%USERPROFILE%\.codeium\windsurf\mcp_config.json`

```json
{
  "mcpServers": {
    "wahooks": {
      "serverUrl": "https://api.wahooks.com/mcp"
    }
  }
}
```

<Callout type="warn">
  Windsurf uses `serverUrl` (not `url`) for remote servers ā€” this differs from other clients.
</Callout>

Restart Windsurf after saving.

## Claude Desktop

Claude Desktop uses stdio-based servers. Use the `mcp-remote` wrapper:

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "wahooks": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://api.wahooks.com/mcp"]
    }
  }
}
```

Fully quit and restart Claude Desktop after editing the config.

## Available Tools

The MCP server exposes 18 tools:

| Category | Tools |
|----------|-------|
| Connections | `list_connections`, `create_connection`, `get_or_create_scannable_connection`, `get_connection`, `delete_connection`, `restart_connection`, `get_qr` |
| Messages | `send_message`, `get_chats`, `get_profile` |
| Webhooks | `list_webhooks`, `create_webhook`, `update_webhook`, `delete_webhook`, `get_webhook_logs`, `test_webhook` |
| Tokens | `list_tokens`, `create_token`, `revoke_token` |

## Troubleshooting

**Authentication fails** ā€” Make sure you have a WAHooks account. Check that pop-ups aren't blocked in your browser.

**Tools not showing up** ā€” Restart your AI assistant after adding the config. Verify the JSON syntax is valid. In Claude Code, run `claude mcp list` to check.

**Windsurf not connecting** ā€” Use `serverUrl` (not `url`) in the config.

**Connection refused** ā€” Verify `https://api.wahooks.com` is accessible from your network.
